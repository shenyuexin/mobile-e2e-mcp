# 阶段切换规则（中文版）

## Phase 1 -> Phase 2

满足以下条件即可进入 Phase 2：

1. iOS 黄金链路达标（样例项目 5 次重复通过率 >= 95%）
2. Android 黄金链路达标（样例项目 5 次重复通过率 >= 95%）
3. 已知系统中断（如权限弹窗、存储密码弹窗、底部 action sheet）已被纳入规则并能处理
4. 失败证据包完整率达标
5. 没有阻断黄金链路的 P0 问题

如果以上满足，则可以认为 **Phase 1 已结束**，不需要额外为了样例项目堆更多重复轮次。

---

## Phase 2 -> Phase 3

满足以下条件即可进入 Phase 3：

1. interruption / retry / fallback / evidence 机制已经可复用
2. Phase 2 引入的稳定性增强没有破坏 Phase 1 黄金链路
3. 已知 P0/P1 稳定性问题有明确处置结论
4. 核心执行引擎已足够稳定，适合扩展到 RN/Flutter profile

如果以上满足，则进入 **Phase 3（框架扩展）** 是合理的。

---

## 核心原则

阶段切换不是看“有没有更多可以优化的点”，而是看：

- 当前阶段目标是否达成
- 阻断当前阶段的关键问题是否已经清理
- 下一个阶段所依赖的基础是否已经稳定

满足这三点，就应该进入下一阶段，而不是无限停留在当前阶段打磨样例项目。
